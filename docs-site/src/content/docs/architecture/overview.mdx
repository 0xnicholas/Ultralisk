Ultralisk 采用分层架构，从底层硬件到上层推理 API，逐步实现统一管控。

## 分层架构

```
┌─────────────────────────────────────────┐
│  推理 API / Ultralisk Core              │
│  鉴权 · 限流 · 内容安全 · 可观测性 · 日志  │
├─────────────────────────────────────────┤
│  平台服务层                              │
│  控制台 · 模型生命周期 · 成本与配额治理     │
├─────────────────────────────────────────┤
│  集群编排层                              │
│  K8s / Slurm · 资源调度 · 弹性伸缩        │
├─────────────────────────────────────────┤
│  硬件与网络层                            │
│  裸金属 · GPU · RDMA · 存储              │
└─────────────────────────────────────────┘
```

## 已交付：Ultralisk Core

Ultralisk Core 是已开源的推理服务接入层，负责：

- **鉴权**：API Key 校验
- **限流**：QPS 限流、token 用量配额
- **内容安全**：输入/输出双侧审核
- **可观测性**：Prometheus 指标
- **日志追踪**：JSON 结构化日志、`request_id`、OTel

它与具体推理引擎（vLLM / TGI / SGLang）解耦，可插拔替换。

## 演进中：平台能力

上层平台能力正在持续开发，包括：

- 统一控制台
- 硬件与网络纳管
- K8s/Slurm 集群编排
- 模型生命周期管理
- 多租户与成本治理

本站点会明确标注每项能力的当前状态。
