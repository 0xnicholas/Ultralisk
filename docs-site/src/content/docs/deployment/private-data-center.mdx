在自有数据中心或私有云中部署 Ultralisk，可以获得完全的数据控制权与合规保障。

## 部署流程

### 1. 评估

- 盘点现有 GPU 服务器、网络、存储
- 确认现有集群编排（K8s / Slurm / 无）
- 明确合规与网络隔离要求

### 2. 设计

- 网络拓扑设计
- 推理引擎选型（vLLM / TGI / SGLang）
- 配额与存储后端设计
- 监控与告警方案

### 3. 部署

- 安装 Ultralisk Core（推理服务接入层）
- 配置 Kong 网关（鉴权、限流、metrics）
- 部署 Prometheus / Grafana / Alertmanager
- （可选）接入现有 K8s/Slurm 集群

### 4. 运维

- 配置日志保留与清理策略
- 配置备份与容灾
- 接入专业服务团队的托管运维支持（可选）

## 最小生产配置

| 组件 | 说明 |
|---|---|
| Ultralisk Core | FastAPI 应用，承载业务逻辑 |
| Kong | API 网关，鉴权 + QPS 限流 |
| Redis | 日 token 配额计数 |
| PostgreSQL | 月 token 配额持久化 |
| Prometheus | 指标抓取 |
| Grafana | 可视化与告警 |
